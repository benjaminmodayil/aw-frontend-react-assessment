# Task 006: Implement Granular Loading States

## Description
Currently, a single global `isLoading` flag disables ALL UI elements when any operation is in progress. This creates a poor user experience during busy periods as users cannot perform non-conflicting operations (e.g., adding a new task while another is being deleted). We need granular loading states that only disable relevant UI elements.

## Technical Analysis
Current issues:
- Global `isLoading` in `src/hooks/useTasks.ts` affects entire UI
- All buttons and inputs disabled during any operation
- Users cannot perform legitimate concurrent operations
- Contributes to the perception of a "laggy" application

## Acceptance Criteria
- [ ] Replace global `isLoading` with operation-specific loading states
- [ ] Only disable UI elements directly involved in active operations
- [ ] Allow non-conflicting concurrent operations
- [ ] Implement per-task loading indicators
- [ ] Maintain form-specific loading state
- [ ] Add loading state for refresh operation
- [ ] Ensure accessibility with proper ARIA states
- [ ] No regression in error handling

## Implementation Notes
1. Loading state structure:
   ```typescript
   interface LoadingState {
     addingTask: boolean;
     togglingTasks: Set<number>; // Task IDs being toggled
     deletingTasks: Set<number>; // Task IDs being deleted
     refreshing: boolean;
     loadingInitial: boolean;
   }
   
   const [loadingState, setLoadingState] = useState<LoadingState>({
     addingTask: false,
     togglingTasks: new Set(),
     deletingTasks: new Set(),
     refreshing: false,
     loadingInitial: true
   });
   ```

2. Update operations to use specific loading states:
   ```typescript
   const toggleTask = useCallback(async (taskId: number) => {
     // Set specific loading state
     setLoadingState(prev => ({
       ...prev,
       togglingTasks: new Set(prev.togglingTasks).add(taskId)
     }));
     
     try {
       // ... perform operation
     } finally {
       // Clear specific loading state
       setLoadingState(prev => {
         const newTogglingTasks = new Set(prev.togglingTasks);
         newTogglingTasks.delete(taskId);
         return { ...prev, togglingTasks: newTogglingTasks };
       });
     }
   }, []);
   ```

3. Update UI components:
   ```typescript
   // TaskItem.tsx
   const TaskItem: React.FC<TaskItemProps> = ({ task, onToggle, onDelete }) => {
     const { loadingState } = useTaskContext();
     const isToggling = loadingState.togglingTasks.has(task.id);
     const isDeleting = loadingState.deletingTasks.has(task.id);
     const isDisabled = isToggling || isDeleting;
     
     return (
       <div className={`task-item ${isDisabled ? 'task-loading' : ''}`}>
         <input
           type="checkbox"
           checked={task.completed}
           onChange={() => onToggle(task.id)}
           disabled={isToggling}
           aria-busy={isToggling}
           aria-label={`Mark "${task.text}" as ${task.completed ? 'incomplete' : 'complete'}`}
         />
         <span className="task-text">
           {task.text}
           {isToggling && <LoadingSpinner size="small" />}
         </span>
         <button
           onClick={() => onDelete(task.id)}
           disabled={isDeleting}
           aria-busy={isDeleting}
         >
           {isDeleting ? <LoadingSpinner size="small" /> : 'Delete'}
         </button>
       </div>
     );
   };
   ```

4. Inline loading indicators:
   ```typescript
   const LoadingSpinner: React.FC<{ size?: 'small' | 'medium' | 'large' }> = ({ 
     size = 'medium' 
   }) => (
     <span 
       className={`loading-spinner loading-spinner--${size}`}
       role="status"
       aria-label="Loading"
     >
       <span className="visually-hidden">Loading...</span>
     </span>
   );
   ```

## Testing Requirements
- Test concurrent operations on different tasks
- Test multiple operations on the same task are queued
- Test loading states are properly cleared on success/failure
- Test accessibility with screen readers
- Verify no operations are blocked unnecessarily

## UX Flow
1. User clicks checkbox on Task A
2. Only Task A's checkbox shows loading spinner
3. User can still interact with Task B, C, etc.
4. User can still add new tasks while Task A is updating
5. User cannot click Task A's checkbox or delete button until operation completes
6. Other users' operations appear with their own loading states
7. Global refresh shows overlay but allows viewing current state
