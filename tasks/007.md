# Task 007: Implement Conflict Resolution System

## Description
Currently, when multiple operations occur on the same data, the last write wins without any conflict detection or resolution. The `refreshTasks` operation simply overwrites the current state, potentially losing in-flight changes. We need a conflict resolution system that can merge changes intelligently and handle cases where the localStorage state differs from the in-memory state.

## Technical Analysis
Current problems:
- No detection of conflicting changes
- Refresh operation overwrites without merging
- No way to recover from split-brain scenarios
- Users report "lost important tasks multiple times"

This is especially important in scenarios where:
- Multiple browser tabs are open
- Browser extensions modify localStorage
- Operations fail partially

2. Merge strategies:
   ```typescript
   class MergeStrategy {
     // Last-write-wins based on timestamp
     mergeByTimestamp(local: Task, remote: Task): Task {
       const localTime = new Date(local.updatedAt || 0).getTime();
       const remoteTime = new Date(remote.updatedAt || 0).getTime();
       return localTime > remoteTime ? local : remote;
     }
     
     // Merge completed status (OR operation - if either is complete, result is complete)
     mergeCompleted(local: Task, remote: Task): Task {
       return {
         ...local,
         completed: local.completed || remote.completed,
         updatedAt: new Date().toISOString()
       };
     }
     
     // Three-way merge with base state
     threeWayMerge(base: Task, local: Task, remote: Task): Task | null {
       // If only one side changed, take that change
       if (this.tasksEqual(base, local)) return remote;
       if (this.tasksEqual(base, remote)) return local;
       
       // Both changed - need conflict resolution
       return null;
     }
   }
   ```
