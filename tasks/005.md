# Task 005: Implement Optimistic Updates with Rollback

## Description
Currently, the UI only updates after async operations complete, causing a laggy user experience. During the 50-350ms delay, users might think their action didn't register and try again, contributing to the race condition issues. We need to implement optimistic updates that immediately reflect changes in the UI, with rollback capability if operations fail.

## Technical Analysis
Current flow:
1. User clicks → UI disabled → Wait 50-350ms → Update state → UI enabled
   
Desired flow:
1. User clicks → Immediately update UI → Async operation in background → Rollback only on failure

This addresses the user's complaint about working during "busy periods" by making the UI feel more responsive.

## Acceptance Criteria
- [ ] All task operations update UI immediately (optimistically)
- [ ] Failed operations roll back to previous state
- [ ] Implement visual indicators for optimistic updates (e.g., opacity/animation)
- [ ] Maintain data consistency between optimistic and confirmed states
- [ ] Handle edge cases (e.g., deleting an optimistically added task)
- [ ] Provide clear feedback when rollbacks occur
- [ ] Ensure localStorage only updates after successful operations

## Implementation Notes
1. Optimistic update pattern:
   ```typescript
   const toggleTask = useCallback((taskId: number) => {
     // Optimistic update
     const previousState = useRef<Task[]>();
     
     setTasks(prev => {
       previousState.current = prev;
       return prev.map(task =>
         task.id === taskId 
           ? { ...task, completed: !task.completed, optimistic: true }
           : task
       );
     });
     
     const operation: QueuedOperation = {
       id: `toggle-${taskId}`,
       type: 'update',
       execute: async () => {
         try {
           const updated = await taskService.updateTask(taskId, {
             completed: !previousState.current?.find(t => t.id === taskId)?.completed
           });
           
           // Confirm optimistic update
           setTasks(prev => prev.map(task =>
             task.id === taskId 
               ? { ...task, ...updated, optimistic: false }
               : task
           ));
           
           await taskService.saveTasks(/* current tasks */);
         } catch (error) {
           throw error; // Triggers rollback
         }
       },
       rollback: () => {
         // Revert to previous state
         if (previousState.current) {
           setTasks(previousState.current);
         }
         // Show error message
         setError(`Failed to update task. Please try again.`);
       }
     };
     
     operationQueue.enqueue(operation);
   }, []);
   ```

2. Visual feedback for optimistic updates:
   ```css
   .task-item.optimistic {
     opacity: 0.7;
     position: relative;
   }
   
   .task-item.optimistic::after {
     content: '';
     position: absolute;
     top: 0;
     right: 0;
     width: 8px;
     height: 8px;

     border-radius: 50%;
     animation: pulse 1s infinite;
   }
   
   @keyframes pulse {
     0% { opacity: 1; }
     50% { opacity: 0.5; }
     100% { opacity: 1; }
   }
   ```

3. Handle complex scenarios:
   ```typescript
   // Deleting an optimistically added task
   const deleteTask = useCallback((taskId: number) => {
     setTasks(prev => {
       const task = prev.find(t => t.id === taskId);
       if (task?.optimistic) {
         // Cancel the add operation instead
         operationQueue.cancelOperation(`add-${taskId}`);
       }
       return prev.filter(t => t.id !== taskId);
     });
     
     // ... rest of delete logic
   }, []);
   ```
